imdb framework

A modern cross-platform C++ framework for (sketch-based) image retrieval

This website describes our modern, cross-platform C++ framework for content-based and sketch-based image retrieval. We use this framework heavily in our own research as well as for teaching the Processing Digital Camera Images class at Technical University of Berlin. The framework's name imdb is short for image database.

Goals

The framework consists of several modules that let you quickly build your own image retrieval application. We have designed the framework with the following objectives in mind:

Flexibility
The framework consists of several modules that can be used independently, letting you easily build applications such as sketch-based image retrieval or 3D shape retrieval.
Efficiency
All code is reasonably efficient - we have used this framework to compute descriptors for 10 million images on a single 8-core machine in about a day.
Modern code
We rely as much as possible on existing libraries (mainly Qt, OpenCV and Boost), modern C++ language features and avoid C-style code.
Cross-platform compatibility
Compiles and runs on OS X, Linux and Windows.
Single machine
The framework does not provide any means to distribute computation over multiple machines, instead we rely on existing solutions (we commonly use Apple XGrid for this).

Compiling

The framework relies on Qt (>=4.7), OpenCV (>=2.2) and Boost (>=1.47), so you need those libraries installed on your system. Our build files assume that your compiler/linker knows the paths to the corresponding header files/libraries (e.g. using an environment variable). Our build process relies on .pro files that are 'compiled' to platform specific project files using the qmake tool that comes with Qt.

OS X

Install MacPorts and run 'sudo port install qt4-mac boost opencv'. By default, MacPorts installs the libraries into /opt/local. Make sure that you have the following two lines in your .profile file in your home directory: export CPLUS_INCLUDE_PATH=/opt/local/include and export LIBRARY_PATH=/opt/local/lib. After restarting your terminal, you should now be able to compile all modules.

Linux

You probably already have all required libraries installed. Otherwise use your favorite package manager, set the include path and linker environment variable and compile.

Windows

You probably need to download and install the required libraries manually. Use qmake to generate a Visual Studio project (default on Windows) and set the include and linker paths in Visual Studio.
